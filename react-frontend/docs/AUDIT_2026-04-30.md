# Free GST Billing Software — Full Audit Report

**Audited version:** v1.6.7
**Date:** 2026-04-30
**Scope:** Full codebase — bugs, correctness, UX friction, safety nets, accessibility
**Findings:** 48 total (10 P0 · 17 P1 · 17 P2 · 4 P3)

Severity legend:
- **P0** ship-blocker (data loss, wrong tax on filed returns, silent broken flow)
- **P1** correctness (wrong output in a specific scenario)
- **P2** safety net / UX friction (imperfect defaults, missing validation)
- **P3** accessibility

## P0 — Ship-blockers

### 1. Silent client field loss when loading a saved client
- **File:** `src/components/InvoiceGenerator.jsx:656` (`selectSavedClient`) and `:685` (`handleClientModalSave`).
- **What breaks:** Both setters build a new `client` object with only `name, address, city, pin, state, gstin`. `country`, `email`, `phone`, and **`isSEZ`** are dropped. Consequence: loading an SEZ client via the auto-complete silently disables the SEZ flag, so `isInterstate = false` for a same-state SEZ deal, giving CGST+SGST instead of IGST. That's a filing / money bug — and the invoice preview loses `client.email`/`client.phone` even though the saved client has them.
- **Fix:** Spread the full saved client: `setClient({ ...cli })`; do the same in `handleClientModalSave`.

### 2. Initial invoice `client` state has no `email/phone/isSEZ`
- **File:** `src/components/InvoiceGenerator.jsx:142`.
- **What breaks:** Default shape is `{ name, address, city, pin, state, gstin, country }`. Even when ClientModal saves those extra fields to the client directory, they never land in `bill.data.client` because they're never in state. Reopening the bill can never re-populate them, and the SEZ toggle set in ClientModal is silently discarded.
- **Fix:** Add `email: '', phone: '', isSEZ: false` to the initial shape.

### 3. Recurring templates use `description` — server reads `name`
- **Files:** `src/components/RecurringInvoices.jsx:17,270-297,137-143` vs `server.js:860,890,898`.
- **What breaks:** A template created via the Recurring page stores items keyed `description`. When the server's `processDueRecurring` fires it, `bill.data.items[*].name` is `undefined`, so the generated bill lists items as `-`. Users open the auto-generated bill and see blank rows.
- **Fix:** Normalize field name to `name` everywhere; or migrate on load.

### 4. Server auto-fire always hardcodes CGST+SGST split (interstate = wrong)
- **File:** `server.js:869-871`.
- **What breaks:** `processDueRecurring` sets `cgst: taxTotal/2, sgst: taxTotal/2, igst: 0` for every recurring invoice regardless of seller state vs client state / SEZ. Interstate retainers ship with wrong tax bucket → GSTR-1 export as intrastate B2B → real filing error.
- **Fix:** Reuse the interstate logic from `InvoiceGenerator` (compare `profile.state` to `tpl.clientState`/`placeOfSupply`; honour `isSEZ`).

### 5. Vite dev proxy points at the wrong port
- **File:** `vite.config.js:152`.
- **What breaks:** `proxy.target = 'http://localhost:3001'`; Express defaults to **47371**. `npm run dev` = every API call 500s. Blocks contributor onboarding.
- **Fix:** Default target to `47371` or read `data/port.txt`.

### 6. `InvoicePreview` interstate detection ignores `placeOfSupply`
- **File:** `src/components/InvoicePreview.jsx:9`.
- **What breaks:** Uses `businessState !== clientState` only. `InvoiceGenerator.jsx:493` correctly honours `details.placeOfSupply` and `isSEZ`. Result: when a user picks a POS different from the client state, totals block shows the InvoiceGenerator-calculated IGST while the item table shows CGST+SGST. Mismatched PDF.
- **Fix:** Mirror the InvoiceGenerator logic, or derive `isInterstate` from `totals.igst > 0`.

### 7. Backup misses several localStorage keys — including ones users notice
- **File:** `src/store.js:271-278`.
- **What breaks:** Whitelist includes `'theme'` (never used) instead of the real key `'freegstbill_theme'`. Also absent: `gst_filing_status` (users lose "GSTR-1 Filed" marks on restore), `freegstbill_dismissedUpdate`, `freegstbill_pwa_dismissed_at`, per-profile `gst_lastUsedAccountId_*`.
- **Fix:** Rename `'theme'` → `'freegstbill_theme'`; add `gst_filing_status` + dismissal keys; wildcard `gst_lastUsedAccountId_*`.

### 8. User-editable invoice number silently overwrites another bill
- **File:** `src/components/InvoiceGenerator.jsx:716-729`; `server.js:102-108`.
- **What breaks:** Server upserts by id. If a user types an invoice-number that already exists (typo, re-sequencing), the previous bill is overwritten with no warning. Data loss.
- **Fix:** Server rejects unless `?overwrite=1`; client warns before save.

### 9. `getNextInvoiceNumber` burns counters on every "New Invoice" click
- **Files:** `src/store.js:102-105`, called from `InvoiceGenerator.jsx:380-383,373,433`.
- **What breaks:** Atomic increment happens the moment the form mounts, before the user types anything. Cancel/close → number is gone. CA-audited businesses require gapless sequences.
- **Fix:** Reserve lazily on first *save*, not mount; or recycle unused numbers when form is abandoned in <30s with no content.

### 10. Receipt-number generation is not atomic
- **File:** `src/components/ReceiptVoucher.jsx:46-51`.
- **What breaks:** `getNextReceiptNo` uses `receipts.length + 1`. Two receipts in quick succession both get `RCP/…/0007`.
- **Fix:** Use the atomic `/meta/counter_RCP/increment` endpoint.

## P1 — Correctness

### 11. GSTR-1 JSON export never emits reverse-charge
- **File:** `src/components/GSTReturns.jsx:896,941`.
- **What breaks:** `rchrg: 'N'` is hardcoded in both `b2b` and `cdnr` blocks even when the bill has `invoiceOptions.reverseCharge = true`. Portal rejects / mis-classifies RCM supplies.
- **Fix:** `rchrg: bill.data?.invoiceOptions?.reverseCharge ? 'Y' : 'N'`.

### 12. GSTR-1 JSON export marks SEZ bills as regular
- **File:** `src/components/GSTReturns.jsx:896`.
- **What breaks:** `inv_typ: 'R'` regardless of `client.isSEZ`. Should be `SEWP` / `SEWOP`. GSTN validators flag on upload.
- **Fix:** Derive from `client.isSEZ`.

### 13. Cess is discarded in every JSON export
- **File:** `src/components/GSTReturns.jsx:895-898,914-916,942-943,954`.
- **What breaks:** Users selling cess-liable HSNs (tobacco, aerated, auto, coal) file GSTR-1 with zero cess. Understatement of liability.
- **Fix:** Aggregate `item.cessPercent * afterDiscount / 100` into `csamt`.

### 14. GSTR-3B JSON zero-fills every table except 3.1(a)
- **File:** `src/components/GSTReturns.jsx:820-837`.
- **What breaks:** Zero-rated exports/SEZ, nil/exempt, RCM inward all emitted as 0.
- **Fix:** Treat SEZ+LUT and delivery-challan invoices as `osup_zero`; count exempt-rate items into `osup_nil_exmp`.

### 15. Expense-tracker ITC always splits CGST+SGST — never IGST
- **File:** `src/components/GSTReturns.jsx:636-640`.
- **What breaks:** Halves `gstAmount` into CGST/SGST unconditionally. But businesses buy AWS/Google/Adobe from out-of-state offices → they charge IGST. Mis-routes ITC in GSTR-3B Table 4(A).
- **Fix:** Add `interstate: bool` to Expense form; route ITC accordingly.

### 16. Purchase bill has no Cess field
- **File:** `src/components/PurchaseBills.jsx:9`.
- **What breaks:** ITC on cess is lost; GSTR-3B `Table 4(A) csamt` stays at zero even when we've paid it.
- **Fix:** Add `cessPercent`; include in `calcItemTax` and route to the ITC block.

### 17. Reverse-charge invoices still charge the buyer for GST
- **Files:** `src/components/InvoiceGenerator.jsx:483-505`; `InvoicePreview.jsx:488-492`.
- **What breaks:** `invoiceOptions.reverseCharge = true` prints the notice but the totals block still *includes* `taxTotal` in the payable. Under Section 9(3)/9(4) the supplier does not collect GST. Users are over-billing then issuing credit notes.
- **Fix:** When `reverseCharge`, zero the tax buckets on the final total; add `rcmTaxAmount` field to `totals` for reporting only.

### 18. Bulk mark-paid / status change never touches payment history
- **File:** `src/components/Dashboard.jsx:154-160,221-240`.
- **What breaks:** Setting status to `paid` sets `paidAmount = totalAmount` but doesn't push into `bill.payments`. Payment history modal shows no payment; ReportsView cashflow doesn't see the payment.
- **Fix:** Push a synthetic payment on bulk mark-paid.

### 19. Receipts don't update the linked invoice
- **File:** `src/components/ReceiptVoucher.jsx:78-93`.
- **What breaks:** Recording a receipt against `againstInvoice: 'INV/…/0007'` saves the receipt but never opens the bill to update `paidAmount`/`payments`. Dashboard still shows the invoice as unpaid.
- **Fix:** On save, look up the bill, append a payment, recompute status.

### 20. Round-off floating point edge case
- **File:** `src/utils.js:945-949`.
- **Fix:** `total = Math.round(total * 100) / 100` first; then `Math.round(total) − total`.

### 21. Duplicate-invoice flow suppresses stock deduction
- **File:** `src/components/InvoiceGenerator.jsx:188`.
- **What breaks:** `stockDeducted.current = !!editingBill`. Duplicating a bill sets `editingBill` so this is `true` — but a duplicate IS a new sale. Stock silently doesn't decrement.
- **Fix:** `stockDeducted.current = !!editingBill && !editingBill._isDuplicate`.

### 22. GSTR-2B reconciliation misses round-off
- **File:** `src/components/GSTReturns.jsx:96-108`.
- **What breaks:** `bookTotals` sums line totals but ignores `purchase.applyRoundOff/roundOff`. Supplier that rounded to ₹1,235 always shows as `amount_mismatch` against 2B `val: 1235`.
- **Fix:** Add round-off to `bookTotals.total` before comparison.

### 23. HSN summary treats B2C-large as tax-exclusive regardless of the bill toggle
- **File:** `src/components/GSTReturns.jsx:922-925`.
- **What breaks:** `computeItemTaxSplit(item, true)` — `taxInclusive` (3rd arg) is omitted, defaulting to `false`. Tax-inclusive B2CL bills inflate taxable and IGST values.
- **Fix:** Pass `!!bill.data?.taxInclusive` as the 3rd arg.

### 24. Profile switch strips `id`, breaking recurring-template resolution
- **File:** `src/App.jsx:265-271`.
- **Fix:** Keep the id; pass it forward.

### 25. Amount-in-words breaks above 10 crore for non-INR
- **File:** `src/components/InvoicePreview.jsx:62-83`.
- **Fix:** Recognise "Billion" branch; share Indian-style Lakh/Crore code where it works.

### 26. `getStateCode` is a hard-coded literal
- **File:** `src/utils.js:225-239`.
- **Fix:** Move to a data file the user can override.

### 27. TDS quarter mapping — **not a bug** (verified fiscal-Q logic correct).

## P2 — Safety nets & UX friction

### 28. Discount is dropped in the RecurringInvoices form
- **File:** `src/components/RecurringInvoices.jsx:270-297`.
- **Fix:** Add the Discount column, or drop the field from the state.

### 29. Purchase-bill tax dropdown is a fixed list — no 0.1% / 0.25% / 3% / custom
- **File:** `src/components/PurchaseBills.jsx:347-354`.
- **Fix:** Reuse the same select+`__custom__` pattern as InvoiceGenerator.

### 30. Purchase-bill Qty has `min="1"`
- **File:** `src/components/PurchaseBills.jsx:337`.
- **Fix:** `min="0" step="any"`.

### 31. Expense-tracker has no round-off
- **File:** `src/components/ExpenseTracker.jsx`.

### 32. `handleBack` uses browser `confirm()` — dangerous default
- **File:** `src/components/InvoiceGenerator.jsx:286-287`.
- **What breaks:** OK = save + exit; Cancel = stay. Users conditioned to "Cancel = discard" hit Cancel expecting to leave. Also `beforeunload` gives a second identical prompt.

### 33. Terms preset picker double-confirms on selection
- **File:** `src/components/InvoiceGenerator.jsx:1712-1716`.
- **Fix:** Ask once per session with "Never ask again" checkbox.

### 34. `handleImageUpload` accepts any file — no MIME/dimension guard

- **Fix:** Whitelist MIME; downscale via canvas above 1024px.

### 35. Naive CSV parser
- **File:** `src/components/InventoryView.jsx:119-131`.
- **Fix:** Handle `""` inside quoted fields.

### 36. No localStorage-size guard
- **File:** `src/store.js:280-294`.
- **Fix:** Emit visible toast on write failure.

### 37. No schema-version tag on backup files
- **File:** `src/store.js:315-334`.
- **Fix:** Add `schemaVersion: 1`; refuse restore of newer major versions.

### 38. `_isDuplicate` leaks into sessionStorage draft
- **File:** `src/App.jsx:285-291`, `InvoiceGenerator.jsx:38-44`.
- **Fix:** Strip after initial run; don't persist those keys.

### 39. Custom unit removal via `__remove__::` value is invisible
- **File:** `src/components/InvoiceGenerator.jsx:1645-1648`.
- **Fix:** Add manage-custom-units modal in Settings.

### 40. Global Ctrl+S / Ctrl+P closure bug
- **File:** `src/components/InvoiceGenerator.jsx:907-923`.
- **Fix:** Add `[isMeaningfulInvoice]` to deps.

### 41. Dashboard `overdue-banner` is a `<div onClick>` — not keyboard-accessible
- **File:** `src/components/Dashboard.jsx:339-357`.
- **Fix:** Make it a `<button>`.

### 42. `getFYOptions` duplicated in 5 files
- **Files:** `Dashboard.jsx:14`, `PurchaseBills.jsx:41`, `ExpenseTracker.jsx:31`, `GSTReturns.jsx:16`, `ReportsView.jsx:12`.
- **Fix:** Export from `utils.js`.

### 43. GST filing status uses different period keys per filter mode
- **File:** `src/components/GSTReturns.jsx:700-706`.
- **Fix:** Normalise to a single canonical period key.

### 44. UPI QR minor edge case around TCS — leaving as note.

## P3 — Accessibility

### 45. Icon-only buttons lack `aria-label`
Everywhere. `title=` alone doesn't work in all screen readers.

### 46. No focus trap or ESC handler on modals
Add a single `<Modal>` component wrapping focus-lock + ESC-to-close.

### 47. `text-muted` (#94a3b8) on white — contrast ratio ≈ 3.2:1 (fails WCAG AA)
Bump to `#64748b` (≈5.6:1).

### 48. Keyboard shortcuts help only reachable via Ctrl+/
Add a small "?" button in sidebar footer.

## Recommended new features

Ranked by impact / effort:

### A. Client Statement PDF — impact 5, effort 2
Single-page "account statement for `<Client>`" — every invoice, credit note, payment, current balance for a date range. #1 daily ask when a client disputes a bill.

### B. Undo / soft-delete for invoices — impact 4, effort 2
Move deleted invoices to `data/bills/_trash/` for 30 days; "Trash" tab in Settings.

### C. Bulk PDF export for a filing period — impact 4, effort 2
"All March invoices as one zip". CAs' #1 request.

### D. Auto-scheduled local backup — impact 3, effort 2
Daily job snapshots `data/*.json` → `data/backups/YYYY-MM-DD.json`, keeps 30.

### E. Invoice-level discount (fixed ₹ or %) — impact 3, effort 2
Retail sellers want "5% off the whole bill".

### F. Client credit limit + warn-on-new-invoice — impact 3, effort 1
Add `creditLimit` to client record; warn banner if new invoice pushes past.
